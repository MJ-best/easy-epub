# Supabase MVP Assumptions

- Agents and skills are workspace-scoped in the MVP.
- `profiles` are auto-created from `auth.users` via trigger after Google OAuth signup.
- `project_goal` is required and non-empty.
- `execution_plan` is stored on `projects` as JSONB and is expected to be generated by the orchestrator.
- Duplicate tasks are prevented with `tasks.dedupe_key` plus a unique constraint on `(project_id, dedupe_key)`.
- Partial artifact generation is supported with `artifacts.status = partial` and `artifacts.is_partial = true`.
- Background orchestration should use `service_role` or server-side functions for agent writes; the SQL policies are scoped for direct client access.
