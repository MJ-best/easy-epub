## ❌ 현재 제한사항 (불가능한 것)

### 1. 완벽한 EPUB 미리보기 불가능
**왜 불가능한가?**

#### 기술적 이유:
1. **Flutter는 EPUB 리더가 아님**
   - EPUB은 전용 리더에서만 정확히 렌더링됨
   - Apple Books, Calibre 등에서만 완벽한 표시

2. **flutter_html의 한계**
   - EPUB 전용 CSS를 100% 지원 안 함
   - 페이지 넘김 구현 불가
   - `@font-face` 사용 불가 (폰트 임베딩 안 됨)

3. **현재 미리보기 vs 실제 EPUB**:
   ```
   미리보기 (flutter_html)
   ├── 기본 HTML 태그만 렌더링
   ├── 시스템 폰트 사용 (Noto Serif KR)
   └── 스크롤 방식

   실제 EPUB (Apple Books 등)
   ├── EPUB 전용 CSS 완벽 적용
   ├── KoPub 폰트 적용 (임베딩되면)
   ├── 페이지 넘김 방식
   └── 리더기별 설정 적용
   ```

**결론**:
- ⚠️ 미리보기는 **참고용**일 뿐
- ✅ 정확한 결과는 **EPUB 리더에서만** 확인 가능

### 2. 폰트 임베딩 미구현
**현재 상태**: ❌ 폰트 파일 포함 안 됨

**CSS에는 있지만**:
```css
font-family: "KoPub Batang", "Noto Serif KR", ...
```

**실제로는**:
- ❌ KoPub 폰트 파일이 EPUB에 포함되지 않음
- ❌ OEBPS/Fonts/ 폴더 생성 안 됨
- ✅ 대신 리더의 기본 폰트 사용

**개선 가능?**: ✅ 가능 (라이선스 확인 필요)

### 3. 이미지 지원 없음
**현재**: ❌ 완전 미지원
- 표지 이미지: 선택 UI 있었지만 제거됨
- 본문 이미지: 마크다운 파싱 안 됨
- 이미지 임베딩: 구현 안 됨

**개선 가능?**: ✅ 가능

### 4. 고급 마크다운 문법
**미지원**:
- ❌ 테이블
- ❌ 인용구 `>`
- ❌ 각주
- ❌ 수학 수식
- ❌ 다이어그램

**개선 가능?**: △ 부분 가능 (복잡함)

## ⚠️ 미리보기 vs 실제 EPUB의 차이

### 미리보기 (앱 내)
```
✅ 빠른 확인용
✅ 대략적인 레이아웃
⚠️ 시스템 폰트만
⚠️ 스크롤 방식
❌ EPUB CSS 일부만
```

### 실제 EPUB (Apple Books)
```
✅ 완벽한 렌더링
✅ 모든 CSS 적용
✅ 페이지 넘김
✅ 리더 설정 반영
✅ 글꼴 크기 조절
```

### 차이점 예시

| 항목 | 미리보기 | 실제 EPUB |
|------|----------|-----------|
| 폰트 | Noto Serif KR | KoPub Batang (임베딩 시) |
| 레이아웃 | 스크롤 | 페이지 넘김 |
| CSS | 기본만 | 완벽 적용 |
| 행간 | 근사치 | 정확함 |
| 여백 | 근사치 | 정확함 |

**중요**:
> 미리보기는 "이런 느낌"만 보여줍니다.
> 정확한 결과는 반드시 EPUB 리더에서 확인하세요!

## 🔧 기술적 한계

### 1. Flutter의 한계
```
Flutter는:
✅ 앱 UI 만들기 - 완벽
❌ EPUB 리더 - 불가능
❌ HTML 완벽 렌더링 - 불가능
```

### 2. flutter_html 패키지의 한계
```
지원:
✅ 기본 HTML 태그 (h1, p, ul, li 등)
✅ 간단한 CSS (color, font-size, margin)
✅ 인라인 스타일

미지원:
❌ @font-face
❌ 복잡한 CSS 셀렉터
❌ EPUB 전용 속성
❌ 페이지 레이아웃
```

### 3. 현실적인 대안
**완벽한 미리보기를 위해서는**:
1. ✅ EPUB 생성 → Apple Books로 열기
2. ❌ 앱 내에서 완벽한 미리보기 (기술적으로 불가능)

## ✅ 개선 가능한 것

### 1. 폰트 임베딩 (우선순위: 높음)
**방법**:
```dart
// 1. 폰트 파일 추가
assets/fonts/
├── KoPubBatang.ttf
└── KoPubDotum.ttf

// 2. EPUB에 복사
OEBPS/Fonts/
├── KoPubBatang.ttf
└── KoPubDotum.ttf

// 3. CSS 업데이트
@font-face {
  font-family: "KoPub Batang";
  src: url("../Fonts/KoPubBatang.ttf");
}
```

**필요사항**:
- KoPub 폰트 라이선스 확인
- EPUB에 폰트 파일 포함 코드 추가

**난이도**: ⭐⭐☆☆☆ (쉬움)

### 2. 이미지 지원
**방법**:
```dart
// 1. 마크다운 이미지 파싱
![alt](image.png) → <img src="../Images/image.png">

// 2. 이미지 파일 EPUB에 포함
OEBPS/Images/
└── image.png

// 3. OPF에 manifest 추가
<item id="img1" href="Images/image.png" media-type="image/png"/>
```

**난이도**: ⭐⭐⭐☆☆ (중간)

### 3. 표지 이미지
**방법**:
```dart
// 1. 이미지 선택
FilePicker → 이미지 파일

// 2. EPUB에 포함
OEBPS/Images/cover.jpg

// 3. OPF에 cover 메타데이터
<meta name="cover" content="cover-image"/>
<item id="cover-image" href="Images/cover.jpg" media-type="image/jpeg"/>
```

**난이도**: ⭐⭐☆☆☆ (쉬움)

### 4. 더 많은 마크다운 문법
**추가 가능**:
- ✅ 인용구 `>`
- ✅ 수평선 `---`
- △ 테이블 (복잡)
- ❌ 각주 (EPUB 구조 변경 필요)

**난이도**: ⭐⭐⭐☆☆ ~ ⭐⭐⭐⭐☆

### 5. EPUB 유효성 검사
**방법**:
```bash
# EPUBCheck 사용
epubcheck book.epub
```

**Flutter 통합**:
```dart
// Process.run으로 실행
await Process.run('epubcheck', [epubPath]);
```

**난이도**: ⭐⭐☆☆☆ (쉬움)

## ❌ 개선 불가능한 것

### 1. 앱 내 완벽한 EPUB 미리보기
**이유**:
- Flutter는 EPUB 리더가 아님
- EPUB 렌더링 엔진 없음
- 페이지 넘김 구현 복잡
- CSS 완벽 지원 불가

**대안**:
- ✅ 현재처럼 flutter_html 사용
- ✅ "참고용 미리보기"로 명시
- ✅ 실제 확인은 EPUB 리더 권장

### 2. 리플로우 없는 고정 레이아웃 EPUB
**현재**: EPUB 2.0 리플로우만 지원

**고정 레이아웃**:
- 만화, 잡지 등
- EPUB 3.0 필요
- 복잡한 구조

**구현 난이도**: ⭐⭐⭐⭐⭐ (매우 어려움)

### 3. DRM 적용
**이유**:
- DRM은 출판사/플랫폼 레벨
- 개인 앱에서 구현 불가
- 라이선스 필요
